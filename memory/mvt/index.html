Memory Allocation in OS

In the MVT scheme, there is no fixed size or partition defined in the memory. Initially, all memory is available for allocation and is considered as a single 'hole'. Processes are allocated memory as and when
they arrive. This prevents any internal fragmentation as there is a more contiguous allocation of memory. As there is no limitation on process size (due to a partition size), the degree of multiprogramming is higher than in MFT and is dynamic. However, external fragmentation can occur as processes may leave holes upon terminating which are too small to be used by a new process.
